## ❌ 不应包含的文件（重要！）

以下文件**绝对不应**打包进 zip（通常很大且可以重新生成）：

- [ ] **`node_modules/` 目录** ⚠️ **必须排除**
  - 根目录下的 `node_modules/`
  - `analyzer-app/node_modules/` 
  - 原因：体积巨大（几百MB到几GB），可以重新安装
- [ ] `analyzer-app/dist/` 目录（构建产物，可以重新生成）
- [ ] `.git/` 目录（Git 历史，如果不需要版本控制）
- [ ] `*.log` 文件（日志文件）
- [ ] `.DS_Store`（Mac 系统文件）
- [ ] `Thumbs.db`（Windows 系统文件）

### ⚠️ 如何排除 node_modules

**方法一：手动删除（推荐，最简单）**

1. **删除根目录的 node_modules**
   - 打开文件资源管理器
   - 找到项目根目录下的 `node_modules` 文件夹
   - 右键点击 → 删除
   - 如果提示"文件路径太长"，使用下面的方法

2. **删除 analyzer-app 目录的 node_modules**
   - 进入 `analyzer-app` 文件夹
   - 找到 `node_modules` 文件夹
   - 右键点击 → 删除

3. **如果删除失败（路径太长）**
   - 使用 7-Zip 或 WinRAR：右键点击 `node_modules` → 7-Zip → 删除
   - 或使用 PowerShell（管理员权限）：
     ```powershell
     Remove-Item -Path "node_modules" -Recurse -Force -ErrorAction SilentlyContinue
     ```
   - 或使用 robocopy（Windows，最可靠）：
     ```cmd
     mkdir empty_folder
     robocopy empty_folder node_modules /MIR
     rmdir empty_folder
     rmdir node_modules
     ```

**方法二：使用压缩工具排除（不删除文件）**

- **Windows 7-Zip**：
  1. 选择项目文件夹
  2. 右键 → 7-Zip → 添加到压缩包
  3. 在"排除"选项中添加：`node_modules\*` 和 `analyzer-app\node_modules\*`
- **Windows WinRAR**：
  1. 选择项目文件夹
  2. 右键 → 添加到压缩文件
  3. 在"文件"选项卡 → "排除文件"中添加：`node_modules\*`
- **Mac 终端**：
  ```bash
  zip -r Analyzer_for_AIE1901-1.zip Analyzer_for_AIE1901-1 \
    -x "*/node_modules/*" "*/dist/*" "*/.git/*"
  ```
- **PowerShell**：
  ```powershell
  Compress-Archive -Path "Analyzer_for_AIE1901-1" `
    -DestinationPath "Analyzer_for_AIE1901-1.zip" `
    -Exclude "node_modules","dist"
  ```

2. 选择"压缩"

## 🔍 最终验证

打包完成后，建议：

1. **解压测试**：
   - 将 zip 文件解压到另一个位置
   - 确认文件结构完整
   - 确认没有遗漏重要文件

2. **文件大小检查**：
   - ⚠️ **如果 zip 文件过大（>50MB），很可能包含了 `node_modules`**
   - ✅ **正常大小应该在 5-20MB 左右**（不含 node_modules）
   - 如果超过 50MB，请检查是否误包含了 node_modules

3. **文档完整性**：
   - 确认 `docs/本地运行指南.md` 和 `docs/快速开始.txt` 存在
   - 确认 `README.md` 已更新并指向正确的文档路径

## ⚠️ 重要提示

1. **必须排除 node_modules**（最重要！）：
   - `node_modules` 文件夹通常很大（几百MB到几GB）
   - 包含后 zip 文件会变得非常大，难以传输
   - 接收者需要重新安装依赖（使用 `pnpm install` 或 `npm install`）
   - 不同操作系统可能需要不同的二进制文件
   - **检查方法**：压缩后如果 zip 文件 > 50MB，很可能包含了 node_modules

2. **保留 package.json**：
   - 这是最重要的文件，包含所有依赖信息
   - 没有它无法安装依赖

3. **提供清晰的文档**：
   - 确保接收者能看到运行指南
   - 在 README.md 中提供快速链接

4. **测试解压**：
   - 在打包后，自己先解压测试一次
   - 确保所有文件都在，结构正确
